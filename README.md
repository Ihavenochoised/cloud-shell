#### ⚙️ Installation

1. Install dependencies

```bash
npm install
npm rebuild
```

2. Generate a self-signed SSL certificate (dev only)
```bash
openssl req -x509 -newkey rsa:2048 -keyout key.pem -out cert.pem -days 365 -nodes -subj "/CN=localhost"
```

> [!NOTE]
> For production, replace these with real certificates from [Let's Encrypt](https://letsencrypt.org/) via Certbot.

3. Generate your bcrypt password hash — run this once and copy the output:
```bash
node -e "import('bcrypt').then(b => b.default.hash('your_password', 12).then(console.log))"
```

4. Create your `.env` file:
```env
PASSWORD_HASH=paste_your_bcrypt_hash_here
JWT_SECRET=replace_this_with_a_long_random_string
```

5. Start the server
```bash
npm start
# → https://localhost:3000
```

Open your browser and go to `https://localhost:3000`.

> [!WARNING]
> Most browsers assume HTTP for localhost, so make sure to type `https://` at the front of the URL!

> [!NOTE]
> On first visit, your browser will warn you about the self-signed certificate. Click **Advanced → Accept the Risk and Continue** (Firefox) or **Advanced → Proceed** (Chrome). This is expected in development.

#### 🚀 Usage

1. Navigate to `https://localhost:3000`
2. Enter your password on the login screen
3. You will be connected to a live shell session in your browser
4. Each browser tab gets its own independent shell session
5. The terminal resizes automatically with your browser window

#### 🏗 Architecture

```
Browser (xterm.js)
      │
      │  wss:// (WebSocket over HTTPS)
      ▼
Node.js + Express (HTTPS)
      │
      │  node-pty
      ▼
PTY (Pseudo Terminal)
      │
      ▼
Shell (bash / cmd.exe)
```

| Layer | Technology | Role |
|---|---|---|
| Browser UI | xterm.js + FitAddon | Renders terminal, captures keystrokes |
| Transport | WebSocket (`wss://`) | Bidirectional, low-latency byte stream |
| Server | Express + ws | Serves static files, handles WS upgrades |
| Shell | node-pty | Spawns a real PTY (`bash` / `cmd.exe`) |

#### 🔒 Security

Authentication is handled in two layers:

1. **Password login** — the `/auth/login` endpoint validates your password using `bcrypt` and returns a short-lived JWT (8 hours)
2. **WebSocket auth** — every WebSocket upgrade request must include a valid JWT as a query parameter (`?token=...`). Connections without a valid token are rejected before a shell is spawned

> [!WARNING]
> This project spawns a real shell on your machine. Only run it on trusted networks or behind a VPN. Never expose it directly to the public internet without additional hardening (rate limiting, IP allowlisting, etc.).
